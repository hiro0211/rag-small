# RAGの技術概要

RAG（Retrieval-Augmented Generation）とは、検索拡張生成と呼ばれる技術です。大規模言語モデル（LLM）の回答精度を向上させるために、外部のナレッジベースから関連情報を検索し、その情報をコンテキストとしてLLMに提供する手法です。

RAGの基本的な処理フローは以下の通りです：
1. ユーザーが質問を入力する
2. 質問文をベクトル化（Embedding）する
3. ベクトルデータベースで類似度検索を行い、関連ドキュメントを取得する
4. 取得したドキュメントをコンテキストとしてLLMのプロンプトに組み込む
5. LLMがコンテキストに基づいて回答を生成する

RAGの利点は、LLMの学習データに含まれない最新情報や社内固有の知識に基づいた回答が可能になることです。また、回答の根拠となるソースドキュメントを提示できるため、回答の信頼性を高めることができます。

# ベクトル検索の仕組み

ベクトル検索（Vector Search）は、テキストや画像などのデータを数値ベクトル（Embedding）に変換し、ベクトル間の距離や類似度を計算して関連するデータを検索する技術です。

テキストのEmbeddingには、OpenAIのtext-embedding-3-smallなどのモデルが使用されます。このモデルはテキストを1536次元のベクトルに変換します。例えば「犬」と「猫」のベクトルは「犬」と「自動車」のベクトルよりも距離が近くなります。

類似度の計算にはコサイン類似度が一般的に使用されます。コサイン類似度は-1から1の値を取り、1に近いほど類似していることを示します。ベクトル検索では、クエリベクトルとデータベース内の全ベクトルとの類似度を計算し、最も類似度の高いものから順に結果を返します。

pgvectorは、PostgreSQLにベクトルデータ型とベクトル検索機能を追加する拡張です。HNSWインデックスを使用することで、高速な近似最近傍探索が可能になります。

# LangChainの基本概念

LangChainは、大規模言語モデル（LLM）を活用したアプリケーション開発のためのフレームワークです。Python版とJavaScript/TypeScript版が提供されています。

LangChainの主要なコンポーネントは以下の通りです：
- **Models**: OpenAI、Anthropicなど各社のLLM APIへの統一的なインターフェース
- **Prompts**: プロンプトテンプレートの管理と動的なプロンプト生成
- **Chains**: 複数のコンポーネントを連結して処理パイプラインを構築
- **Memory**: 会話履歴の管理と文脈の保持
- **Retrieval**: ドキュメントの読み込み、チャンク分割、ベクトル検索

LangChainを使用することで、RAGシステムの構築が大幅に簡略化されます。ドキュメントローダーでPDFやテキストファイルを読み込み、テキストスプリッターでチャンク分割し、Embeddingモデルでベクトル化し、ベクトルストアに格納するという一連の処理を、少ないコードで実装できます。
